Movie API Documentation

This API provides access to movie data and user management functionality. Below are the available endpoints:

Business Logic HTTP Method URL Request Body Response Body
Return a list of ALL movies to the user GET /movies None JSON array:
[
  {
    "title": "string",
    "description": "string",
    "genre": "string",
    "director": "string",
    "imageURL": "string",
    "featured": "boolean"
  },
  ...
]
Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user GET /movies/[title] None JSON object:
{
  "title": "string",
  "description": "string",
  "genre": "string",
  "director": "string",
  "imageURL": "string",
  "featured": "boolean"
}
Return data about a genre (description) by name/title (e.g., "Thriller") GET /genres/[name] None JSON object:
{
  "name": "string",
  "description": "string"
}
Return data about a director (bio, birth year, death year) by name GET /directors/[name] None JSON object:
{
  "name": "string",
  "bio": "string",
  "birthYear": "string",
  "deathYear": "string"
}
Allow new users to register POST /users JSON object:
{
  "username": "string",
  "password": "string",
  "email": "string",
  "birthday": "date"
}		 JSON object:
{
  "id": "string",
  "username": "string",
  "email": "string",
  "birthday": "date",
  "favoriteMovies": []
}
Allow users to update their user info (username) PUT /users/[unsername] None JSON object:
{
  "id": "string",
  "username": "string",
  "email": "string",
  "birthday": "date",
  "favoriteMovies": ["array"]
}
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) PUT /users/[username]/movies/[movieId] None Text message:
"Movie has been added to favorites"
Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later) DELETE /users/[username]/movies/[movieId] None Text message:
"Movie has been removed from favorites"
Allow existing users to deregister (showing only a text that a user email has been removed—more on this later) DELETE /users/[username] None Text message:
"User [email] has been removed"

Notes

  • [title] - Replace with the actual movie title
  • [name] - Replace with the actual genre or director name
  • [username] - Replace with the actual username
  • [movieId] - Replace with the actual movie ID
  • All JSON responses will include appropriate HTTP status codes (200, 201, 400, 404, 500)
  • For POST and PUT requests, include the JSON object in the request body with Content-Type: application/json
  • Error responses will include appropriate HTTP status codes and error messages in JSON format
